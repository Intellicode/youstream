# Commands

- `deno check`
- `deno fmt`
- `deno run tests`

# Workflow

1. Create a new branch
2. Implement the feature
3. Write tests
4. Run `deno check`
5. Run `deno fmt`
6. Run `deno run tests`
7. Create a pull request

# Stack

- Deno
- Hono
- Tailwind

# Best practices

- Use Hono/jsx for UI rendering.
- Use Tailwind for styling.
- Use Deno std for as much as possible
- Do not include extra dependencies unless it saves more than 50 lines of code
- Write tests
